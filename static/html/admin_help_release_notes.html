Release notes

v5.0.2

2024-05-27

  • Improved "AudiencePlayer User Synchronisation" for latest WordPress version 6.5+.

v5.0.1

2024-04-23

  • Updated the AudiencePlayer Embed Player dependency to the latest version, with improved synchronisation of Asset Time Markers.

v5.0.0

2024-04-11

  • Updated the AudiencePlayer Embed Player dependency to the latest version.
  • Ensure you upgrade the AudiencePlayer Plugin to version 5.0.0 or higher before May 1st 2024. From this date lower versions may lose compatibility with the AudiencePlayer streaming infrastructure. For more information, please see CDN migration on May 1, 2024.

v4.6.1

2024-03-11

  • When switching subscriptions on the my-account page, the listed subscription price is now the price per installment instead of the initial price (since the "price per installment" is the actual purchase price in this case).

v4.6.0

2023-08-01

  • The "article-detail" shortcode has been expanded with configuration flag "show_seasons", so that a User can select the season for which to show the episodes.

v4.5.0

2023-05-24

  • Added subscription-switch feature, which allows Users to switch their current subscription plan. This new feature is introduced in the existing shortcode "audienceplayer-shortcode-user-account.php", within the "subscription_info" block. See updated plugin help section for this shortcode with new configuration options.

2023-04-11

v4.2.2

2023-03-27

  • Updated dependency "AudiencePlayer Embed Player", which improves time tracking accuracy during the last 30 seconds of playback.

v4.2.1

2023-03-16

  • Added a new translation "dialogue_voucher_code_applicable_subscription_unknown" for the use case when a voucher code can only be claimed on a new or existing subscription.

v4.2.0

2023-03-06

  • Improved automatic subscription/product purchase popup flow to avoid query-string parameter clashes for "purchase_subscription_id" and "purchase_product_id" when the page is reloaded.
    (N.B. helper method "removeQsaParameters" in audienceplayer-core.js, was renamed to "removeQsaParametersFromString")
  • Added "retry"-button to the "failed payment-order validation" modal, when returning to site from payment provider (e.g. Mollie), so that the User may retry the validation. Included additional logging statement.
  • Improved Admin logs table with info-button to immediately inspect the log properties (no longer necessary to first download the Excel export to inspect the properties).
  • Routine composer library dependency update.

v4.1.1

2023-02-03

  • Updated library dependency AudiencePlayer API client for PHP (v2.0.1: fix for custom integrations on Admin scope).

v4.1.0

2023-02-01

  • Fixed minor compatibility issues for older PHP-versions 7.1 and 7.2 (N.B. If you are still on PHP v7.x, please consider upgrading to v8.x whenever possible!).
  • Improved debugging capability (raw return) and metadata sanitisation (escaped quotes) on method "syncWordpressUserToAudiencePlayer" (often used in custom integrations).

v4.0.0

2022-12-01

  • Updated AudiencePlayer API wrapper dependency to new major version 2
    Warning - possible breaking change depending on your custom implementation: Direct scripted usage of this dependency or custom references via this plugin to "$AudiencePlayerWordpressPlugin->api()" (IF any), should be checked.
  • Improved error logging of AudiencePlayer API-calls.
  • Plugin configuration tab "General" has been renamed to "Integrations".
  • Introduced helper methods for the entitlement of Products and Subscriptions via Wordpress (See Help > Code Examples) + integration for WooCommerce which implements these helper methods automatically (available as "bèta feature" in "Configuration > Integrations").
